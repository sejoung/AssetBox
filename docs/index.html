Usage Guide

모델 불러오기 Solid Wireframe Normals Normal Map UV Retopo 검증 패널 리포트 / 썸네일 워크플로우

1. 모델 불러오기

이럴 때: 모델러가 보내준 에셋을 빠르게 확인 / DCC 툴 없이 미리보기 / 여러 에셋 순차 검수

  1. 파일을 AssetBox 창으로 드래그 앤 드롭합니다.
  2. 모델이 자동으로 로드되고 뷰포트 중앙에 배치됩니다.
  3. 마우스: 회전(좌클릭), 줌(스크롤), 패닝(우클릭)
Tip: 모델을 놓쳤을 때 F 키로 모델 중심으로 카메라 복귀. 한글 경로, 공백 포함 경로도 지원.

2. Solid 뷰 1

이럴 때: 전체 외관 확인 / 내장 텍스처 적용 확인 / 실루엣 검토

  • 메시에 구멍이나 찢어진 부분이 없는지 확인
  • 실루엣이 자연스러운지, 스케일이 일관적인지 검토
배경색 전환: 하단 색상 원 클릭. 어두운 배경=밝은 에셋, 밝은 배경=어두운 에셋/실루엣

3. Wireframe 뷰 2

이럴 때: 토폴로지 흐름(edge flow) 확인 / 과밀 영역 탐색 / 내부 지오메트리 감지

  • 엣지 플로우: 관절, 얼굴 주변 루프가 자연스러운지
  • 불필요한 밀집: 특정 영역에 폴리곤이 과도하게 몰려있지 않은지
  • 내부 면: 보이지 않는 곳에 불필요한 면이 숨겨져 있지 않은지
Tip: Retopo 뷰에서 과밀 표시된 영역을 Wireframe에서 다시 확인하면 원인 파악이 쉽습니다.

4. Normals 뷰 3

이럴 때: 엔진에서 면이 검게 보일 때 / 라이팅 이상 원인 파악 / 노말 품질 최종 점검

파란 점 (작은): 정상 노말
빨간 점 (큰): 뒤집힌 노말 — 면이 안쪽을 향함

뒤집힌 노말 발견 시 하단 "Flipped Normals" 버튼 → 클릭하면 해당 위치로 카메라 이동

수정: Maya: Mesh Display > Reverse Normals / Blender: Mesh > Normals > Flip / 3ds Max: Edit Normals > Flip

5. Normal Map 뷰 4

이럴 때: 노말 방향 분포 확인 / 베이크 전 기본 노말 상태 확인 / 스무딩 그룹 검토

빨간: X축(좌우) 초록: Y축(상하) 파란/보라: Z축(앞뒤)
  • 같은 평면에서 색상이 급변하면 스무딩 문제
  • 하드 엣지 경계에서 색상 구분은 정상

6. UV 뷰 5

이럴 때: 텍스처 매핑 확인 / UV 스트레칭 감지 / 텍셀 밀도 균일성 확인

  • 정사각형 균일: UV가 잘 펼쳐짐, 텍셀 밀도 균일
  • 늘어난 사각형: UV 스트레칭 → 텍스처가 뭉개짐
  • 크기 차이 큼: 텍셀 밀도 불균일
Tip: 중요 부위(얼굴, 무기)의 체커가 더 작아야(밀도 높아야) 합니다. 시임은 안 보이는 곳에 배치.

7. Retopo 뷰 6

이럴 때: 스캔 데이터 품질 평가 / 리토폴로지 필요성 판단 / 납품 전 메시 품질 점검

과밀 (불필요한 폴리곤) 적정 과소 (디테일 부족) 얇은 삼각형
항목양호문제
얇은 삼각형< 5%> 5% — 텍스처 스트레칭/라이팅 아티팩트
과밀 영역< 10%> 10% — 렌더링 자원 낭비
과소 영역< 10%> 10% — 디테일 부족
밀도 편차< 100x> 1000x — 극단적 불균일
판정: "Topology OK"(초록) = 사용 가능 / "Retopology Recommended"(빨간) = 리토폴로지 권장

8. 검증 패널 활용

이럴 때: 프로젝트 기준 충족 확인 / QA 검수 / 아웃소싱 인수

9. 리포트 & 썸네일

Report 버튼: HTML 검증 보고서 생성 → 모델러에게 수정사항 전달, 품질 이력 관리에 활용

Thumbnail 버튼: PNG 스크린샷 저장 → 에셋 라이브러리 카탈로그, 문서 첨부에 활용

Tip: 그리드는 캡처에서 자동 제외. 밝은 배경 전환 시 문서 삽입용으로 깔끔. 어떤 뷰 모드에서든 캡처 가능.

10. 실무 워크플로우

에셋 납품 검수 (QA)

  1. 에셋 드래그 앤 드롭 → 종합 판정 확인
  2. 빨간/노란 항목 상세 확인 (Geometry, Topology)
  3. Normals 뷰(3)로 뒤집힌 노말 → 포커스 버튼으로 위치 파악
  4. UV 뷰(5)로 텍셀 밀도, Retopo 뷰(6)로 메시 품질
  5. Report 생성 → 모델러에게 전달

모델링 중간 점검

  1. DCC 툴에서 FBX/GLB 익스포트 → AssetBox 드롭
  2. Wireframe(2) → Normals(3) → UV(5) → Retopo(6) 순서로 점검
  3. 문제 발견 시 DCC 툴로 돌아가 수정 후 재익스포트

스캔 데이터 평가

  1. 스캔 메시 드롭 → Retopo 뷰(6) 진단 확인
  2. "Retopology Recommended"이면 히트맵으로 문제 영역 파악
  3. 리토폴로지 도구 처리 후 다시 AssetBox에서 개선 확인
Loading Models Solid Wireframe Normals Normal Map UV Retopo Validation Report / Thumbnail Workflows

1. Loading Models

When: Quickly reviewing assets / previewing without DCC tools / batch inspection

  1. Drag and drop a file onto the AssetBox window.
  2. Model loads automatically and centers in the viewport.
  3. Mouse: rotate (left-click), zoom (scroll), pan (right-click)
Tip: Press F to refocus on the model. Unicode paths and spaces are supported.

2. Solid View 1

When: Checking overall appearance / verifying embedded textures / silhouette review

  • Check for holes or tears in the mesh
  • Verify natural silhouette and consistent scale
Background: Click color circles in toolbar. Dark=light assets, Light=dark assets/silhouette

3. Wireframe View 2

When: Checking edge flow / finding dense areas / detecting internal geometry

  • Edge flow: Natural loops around joints and faces
  • Density: No excessive polygon concentration
  • Internal faces: No hidden unnecessary geometry
Tip: Cross-reference areas flagged in Retopo view with Wireframe to understand the root cause.

4. Normals View 3

When: Black faces in engine / abnormal lighting / final normal quality check

Blue (small): Correct normals
Red (large): Flipped normals — faces point inward

When flipped normals found → "Flipped Normals" button → click to fly camera to location

Fix: Maya: Mesh Display > Reverse Normals / Blender: Mesh > Normals > Flip / 3ds Max: Edit Normals > Flip

5. Normal Map View 4

When: Inspecting normal direction distribution / pre-bake check / smoothing group review

Red: X-axis Green: Y-axis Blue/Purple: Z-axis
  • Abrupt color changes on flat surfaces = smoothing issues
  • Distinct color separation at hard edges is normal

6. UV View 5

When: Verifying texture mapping / detecting UV stretching / checking texel density

  • Uniform squares: Well-unwrapped UVs, consistent texel density
  • Stretched rectangles: UV stretching → blurry textures
  • Varying sizes: Uneven texel density
Tip: High-importance areas (face, weapon) should have smaller checkers. Place seams in hidden areas.

7. Retopo View 6

When: Evaluating scan data / deciding retopology need / pre-delivery mesh check

Over-dense Optimal Under-dense Thin triangles
MetricGoodBad
Thin triangles< 5%> 5% — stretching/lighting artifacts
Over-dense< 10%> 10% — wasted resources
Under-dense< 10%> 10% — lacking detail
Density ratio< 100x> 1000x — extremely uneven
Verdict: "Topology OK" (green) = usable / "Retopology Recommended" (red) = needs work

8. Validation Panel

When: Checking project standards / QA inspection / receiving outsourced assets

CategoryWatch ForAction
GeometryTris/Verts in redLODs or polygon reduction
TopologyNon-manifold, FlippedClean up in DCC, flip normals
UVNo UVsUV unwrapping required
TextureMissing in redCheck delivery or naming convention
MaterialNo MaterialVerify material assignment
TransformNon-uniform ScaleApply Freeze Transform

9. Report & Thumbnail

Report: HTML validation report → share with modelers for fix requests, maintain quality records

Thumbnail: PNG screenshot → asset library catalogs, document attachments

Tip: Grid auto-excluded from capture. Light background = cleaner for documents. Works in any view mode.

10. Workflow Examples

Asset Delivery QA

  1. Drop asset → check overall verdict
  2. Review red/yellow items (Geometry, Topology)
  3. Normals view (3) → flipped normals → focus button to locate
  4. UV view (5) texel density, Retopo view (6) mesh quality
  5. Generate Report → send to modeler

Mid-Production Check

  1. Export FBX/GLB from DCC → drop into AssetBox
  2. Wireframe (2) → Normals (3) → UV (5) → Retopo (6)
  3. Fix issues in DCC, re-export, re-check

Scan Data Evaluation

  1. Drop scanned mesh → Retopo view (6)
  2. If "Retopology Recommended" → review heatmap for problem areas
  3. Process with retopo tools, re-check in AssetBox

Installation Guide

AssetBox is an open-source project and is not code-signed. Your OS may show a security warning on first launch.

  1. Download the .dmg file and open it
  2. Drag AssetBox to the Applications folder
  3. Open Terminal and run:
xattr -cr /Applications/AssetBox.app
  1. Now open AssetBox normally from Applications
Why? macOS blocks unsigned apps with "damaged" error. The command above removes the quarantine flag. This is safe for open-source apps you trust.
  1. Download the .msi file and run it
  2. "Windows protected your PC" (SmartScreen) will appear
  3. Click "More info"
  4. Click "Run anyway"
  5. Follow the installer. This warning only appears once.
Alternative: Download the .exe file for a portable version (no install needed).
