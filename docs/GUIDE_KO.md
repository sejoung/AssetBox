## 1. 모델 불러오기

### 이럴 때 사용하세요
- 모델러가 보내준 에셋을 빠르게 확인할 때
- DCC 툴(Maya, Blender, 3ds Max)을 열지 않고 간단히 미리보기할 때
- 여러 에셋을 순차적으로 빠르게 검수할 때

### 사용 방법
1. 파일을 AssetBox 창으로 드래그 앤 드롭합니다.
2. 모델이 자동으로 로드되고 뷰포트 중앙에 배치됩니다.
3. 마우스로 회전(좌클릭 드래그), 줌(스크롤), 패닝(우클릭 드래그)할 수 있습니다.

### 팁
- 모델을 놓쳤을 때 `F` 키를 누르면 모델 중심으로 카메라가 돌아옵니다.
- 한글 경로, 공백이 포함된 경로도 정상 동작합니다.
- 새 파일을 드롭하면 이전 모델이 자동으로 교체됩니다.

## 2. Solid 뷰 활용

**단축키: `1`**

### 이럴 때 사용하세요
- 모델의 전체적인 외관을 확인할 때
- 내장 텍스처가 제대로 적용되었는지 확인할 때
- 실루엣과 형태가 의도한 대로인지 검토할 때

### 확인 포인트
- 메시에 구멍이나 찢어진 부분이 보이지 않는지
- 전체 실루엣이 자연스러운지
- 스케일이 다른 에셋과 일관성이 있는지

### 배경색 전환
하단 툴바의 색상 원을 클릭해서 배경을 바꿔보세요.
- **어두운 배경:** 밝은 색 에셋 확인에 적합
- **밝은 배경:** 어두운 색 에셋이나 실루엣 확인에 적합
- **중간 배경:** 범용적인 확인에 적합

## 3. Wireframe 뷰 활용

**단축키: `2`**

### 이럴 때 사용하세요
- 메시의 토폴로지 흐름(edge flow)을 확인할 때
- 불필요하게 폴리곤이 집중된 영역을 찾을 때
- 삼각형화(triangulation) 결과를 확인할 때
- 숨겨진 내부 지오메트리가 있는지 확인할 때

### 확인 포인트
- **엣지 플로우:** 캐릭터 모델의 경우 관절, 얼굴 주변의 루프가 자연스러운지
- **불필요한 밀집:** 특정 영역에 과도하게 폴리곤이 몰려있지 않은지
- **내부 면:** 보이지 않는 곳에 불필요한 면이 숨겨져 있지 않은지
- **N-gon 흔적:** 삼각형화 후 비정상적인 패턴이 보이지 않는지

### 팁
다른 뷰 모드에서 확인한 문제 영역을 Wireframe에서 다시 확인하면 원인을 파악하기 쉽습니다. 예를 들어 Retopo 뷰에서 과밀로 표시된 영역이 실제로 어떤 토폴로지를 가지는지 Wireframe에서 확인할 수 있습니다.

## 4. Normals 뷰 활용

**단축키: `3`**

### 이럴 때 사용하세요
- 모델을 엔진에 임포트했더니 일부 면이 검게 보일 때
- 라이팅이 이상하게 적용되는 원인을 찾을 때
- 아웃라인 셰이더가 비정상적으로 동작할 때
- 모델 납품 전 노말 품질을 최종 점검할 때

### 읽는 방법
- **파란 점 (작은 크기):** 정상 노말. 면의 바깥쪽을 향하고 있습니다.
- **빨간 점 (큰 크기):** 뒤집힌 노말. 면이 안쪽을 향해 렌더링 시 보이지 않거나 라이팅이 반전됩니다.

### "Flipped Normals" 포커스 버튼
뒤집힌 노말이 발견되면 하단 중앙에 빨간 버튼이 나타납니다.
- 버튼에 뒤집힌 버텍스 수가 표시됩니다.
- 클릭하면 카메라가 해당 위치로 자동 이동합니다.
- 복잡한 모델에서 문제 위치를 빠르게 찾을 때 유용합니다.

### 수정 방법
DCC 툴에서 해당 면을 선택하고 노말을 뒤집어 주세요:
- **Maya:** Mesh Display > Reverse Normals
- **Blender:** Mesh > Normals > Flip
- **3ds Max:** Edit Normals > Flip

## 5. Normal Map 뷰 활용

**단축키: `4`**

### 이럴 때 사용하세요
- 버텍스 노말의 방향 분포를 색상으로 시각적으로 확인할 때
- 하이폴리에서 로우폴리로 노말을 베이크하기 전에 기본 노말 상태를 확인할 때
- 노말 스무딩 그룹이 의도대로 설정되었는지 확인할 때

### 읽는 방법
노말 방향이 RGB 색상으로 매핑됩니다:
- **빨간색 계열:** X축(좌우) 방향의 노말이 강한 면
- **초록색 계열:** Y축(상하) 방향의 노말이 강한 면
- **파란색/보라색 계열:** Z축(앞뒤, 카메라 방향) 노말이 강한 면

### 확인 포인트
- 같은 평면인데 색상이 급변하면 노말 스무딩 문제가 있을 수 있습니다.
- 비슷한 방향의 면은 비슷한 색상이어야 합니다.
- 하드 엣지(스무딩 그룹 경계)에서는 색상이 뚜렷하게 구분되는 것이 정상입니다.

## 6. UV 뷰 활용

**단축키: `5`**

### 이럴 때 사용하세요
- 텍스처가 메시에 제대로 매핑되는지 확인할 때
- UV 스트레칭이나 왜곡이 있는지 확인할 때
- UV 시임(이음새) 위치가 적절한지 확인할 때
- 체커 패턴으로 텍셀 밀도의 균일성을 확인할 때

### 읽는 방법
빨간/남색 체크무늬 패턴이 메시 위에 적용됩니다:
- **정사각형이 균일하게 보이면:** UV가 잘 펼쳐져 있고 텍셀 밀도가 균일합니다.
- **사각형이 늘어나 보이면:** UV 스트레칭이 있습니다. 텍스처가 뭉개져 보일 수 있습니다.
- **사각형이 찌그러져 보이면:** UV 왜곡이 있습니다.
- **사각형 크기가 크게 다르면:** 텍셀 밀도가 불균일합니다.

### 확인 포인트
- 카메라에 가까운(중요한) 부위의 체커가 더 작아야(밀도가 높아야) 합니다.
- 캐릭터 얼굴, 무기 같은 주요 에셋은 체커 크기가 균일해야 합니다.
- UV 시임이 눈에 띄지 않는 곳(접히는 부분, 뒷면 등)에 있는지 확인합니다.

## 7. Retopo 뷰 활용

**단축키: `6`**

### 이럴 때 사용하세요
- 스캔 데이터나 자동 리메시 결과의 품질을 평가할 때
- 리토폴로지가 필요한지 판단할 때
- 폴리곤 최적화가 필요한 영역을 찾을 때
- 모델 납품 전 메시 품질을 전체적으로 점검할 때

### 히트맵 읽는 방법
메시가 삼각형 밀도에 따라 색상으로 표시됩니다:

| 색상 | 의미 | 조치 |
|------|------|------|
| 파란색 | 과밀 — 불필요하게 폴리곤이 많은 영역 | 폴리곤 감소(Decimate) 또는 수동 리토폴로지 고려 |
| 초록색 | 적정 밀도 — 양호한 상태 | 조치 불필요 |
| 빨간색 | 과소 — 디테일이 부족한 영역 | 서브디비전 또는 수동으로 디테일 추가 고려 |
| 노란색 | 얇은 삼각형 — 형태가 나쁜 삼각형 | 라이팅/텍스처 아티팩트의 원인. 리토폴로지 필요 |

### 진단 패널 읽는 방법
상단 패널에 표시되는 수치:

| 항목 | 좋은 상태 | 나쁜 상태 |
|------|-----------|-----------|
| 얇은 삼각형 | < 5% | > 5% — 텍스처 스트레칭과 라이팅 아티팩트 유발 |
| 과밀 영역 | < 10% | > 10% — 렌더링 자원 낭비 |
| 과소 영역 | < 10% | > 10% — 디테일 부족 |
| 밀도 편차 | < 100x | > 1000x — 밀도가 극단적으로 불균일 |

### 실무 판단 기준
- **"Topology OK" (초록):** 현재 토폴로지로 사용 가능합니다.
- **"Retopology Recommended" (빨간):** 문제가 있으며, 사유 목록을 확인하세요.
  - 게임용 에셋이면 리토폴로지를 강력히 권장합니다.
  - 시네마틱/렌더용이면 상황에 따라 무시할 수 있습니다.

## 8. 검증 패널 활용

### 이럴 때 사용하세요
- 에셋이 프로젝트 기준에 맞는지 한눈에 확인할 때
- QA 팀에서 에셋 검수 시
- 외부 아웃소싱 에셋을 인수받을 때

### 패널 구조
우측 패널에 6개 카테고리로 정리된 검증 결과가 표시됩니다.

### 카테고리별 실무 활용

#### Geometry
- **Tris/Verts가 빨간색:** 엔진 퍼포먼스에 직접 영향. LOD가 필요하거나 폴리곤 감소 필요.
- **File Size가 노란색 이상:** 로딩 시간과 메모리에 영향. 텍스처 해상도 줄이기나 메시 최적화 필요.
- **Degenerate Tris:** 면적 0인 삼각형은 GPU 자원 낭비. DCC 툴에서 정리 필요.

#### Topology
- **Non-manifold이 빨간색:** 3D 프린팅이나 물리 시뮬레이션에 문제. Boolean 연산 잔재일 가능성 높음.
- **Open Edges:** 메시가 닫혀있지 않음. 투명도 문제나 그림자 아티팩트 유발 가능.
- **Flipped Normals:** Normals 뷰(`3`)에서 빨간 점으로 확인하고 DCC 툴에서 수정.

#### UV
- **No UVs가 빨간색:** 텍스처를 적용할 수 없음. 반드시 UV 펼치기 필요.
- **UV Channels:** 라이트맵에 2번째 채널이 필요한 엔진(Unity, Unreal)에서는 2개 이상이어야 함.

#### Texture
- **Missing이 빨간색:** 필요한 텍스처가 누락됨. 파일 전달 누락이거나 네이밍 규칙 불일치.
- **Max Resolution이 노란색:** 모바일은 2048px 이하, PC/콘솔은 4096px 이하 권장.

#### Material
- **No Material:** 머티리얼 미할당 메시. 엔진에서 기본 셰이더로 표시되므로 의도한 것인지 확인.

#### Scale/Transform
- **Non-uniform Scale:** 라이팅과 물리 연산에 문제. DCC 툴에서 Freeze Transform 필요.
- **Pivot Offset:** 엔진 임포트 시 위치가 어긋남. 피벗을 원점으로 재설정 필요.

## 9. 리포트 & 썸네일 활용

### 검증 리포트

#### 이럴 때 사용하세요
- 에셋 검수 결과를 문서화할 때
- 모델러에게 수정 사항을 전달할 때
- 프로젝트 에셋 품질 이력을 관리할 때

#### 활용 방법
1. 모델을 로드한 후 우측 하단의 "Report" 버튼을 클릭합니다.
2. 모델 파일 옆에 `_report.html` 파일이 생성됩니다.
3. 브라우저에서 열어 검토하거나, 팀에 공유합니다.

#### 리포트에 포함되는 내용
- 파일 정보 (포맷, 파일명, 종합 판정)
- 카테고리별 검증 결과 + 각 항목 설명
- 텍스처 파일 목록 및 누락 텍스처
- 리토폴로지 진단 결과
- 인쇄 가능한 레이아웃

### 썸네일

#### 이럴 때 사용하세요
- 에셋 라이브러리 카탈로그 이미지로 사용할 때
- 에셋 관리 툴(Perforce, Shotgrid 등)에 미리보기 이미지를 등록할 때
- 문서나 슬라이드에 에셋 이미지를 첨부할 때

#### 활용 방법
1. 원하는 뷰 모드와 각도로 모델을 배치합니다.
2. 우측 하단의 "Thumbnail" 버튼을 클릭합니다.
3. 모델 파일 옆에 `_thumbnail.png` 파일이 생성됩니다.

#### 팁
- 그리드는 자동으로 캡처에서 제외됩니다.
- 밝은 배경으로 전환하면 문서 삽입용으로 더 깔끔합니다.
- Wireframe이나 UV 뷰 상태에서도 캡처 가능합니다.

## 10. 실무 워크플로우 예시

### 에셋 납품 검수 (QA)

```
1. 에셋 파일을 드래그 앤 드롭
2. 우측 패널에서 종합 판정 확인 (Good/Warning/Bad)
3. 빨간색/노란색 항목 중심으로 상세 확인:
   - Geometry → 폴리곤/파일 크기 기준 충족 여부
   - Topology → Non-manifold, Flipped Normals 확인
4. Normals 뷰(3)에서 뒤집힌 노말 확인 → 포커스 버튼으로 위치 파악
5. UV 뷰(5)에서 텍셀 밀도 균일성 확인
6. Retopo 뷰(6)에서 메시 품질 진단
7. Report 버튼으로 리포트 생성 → 모델러에게 전달
```

### 아웃소싱 에셋 인수

```
1. 에셋 파일을 드래그 앤 드롭
2. Solid 뷰(1)에서 전체 외관 확인
3. Wireframe 뷰(2)에서 토폴로지 품질 확인
4. 우측 패널에서 프로젝트 기준 충족 여부 확인:
   - Tris < 프로젝트 기준치
   - 텍스처 누락 없음
   - 스케일 균일
5. 문제 없으면 Thumbnail 저장 → 에셋 라이브러리에 등록
6. 문제 있으면 Report 생성 → 수정 요청
```

### 모델링 작업 중간 점검

```
1. DCC 툴에서 FBX/GLB 익스포트
2. AssetBox에 드래그 앤 드롭
3. Wireframe 뷰(2)로 엣지 플로우 확인
4. Normals 뷰(3)으로 노말 문제 체크
5. UV 뷰(5)로 UV 펼치기 상태 확인
6. Retopo 뷰(6)로 밀도 분포 체크
7. 문제 발견 시 DCC 툴로 돌아가 수정 후 재익스포트
```
