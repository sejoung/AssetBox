## 1. 개요

AssetBox는 게임 및 실시간 3D 콘텐츠 제작 파이프라인에서 3D 에셋의 품질을 빠르게 미리보기하고 검증하는 데스크톱 애플리케이션입니다.

- **프레임워크:** Tauri 2 (Rust 백엔드 + React 프론트엔드)
- **3D 렌더링:** Three.js + React Three Fiber
- **플랫폼:** Windows, macOS, Linux
- **기본 창 크기:** 1280 x 800px (최소 900 x 600px)

## 2. 지원 파일 포맷

### 3D 모델 입력

| 포맷 | 확장자 | 로더 |
|------|--------|------|
| FBX | `.fbx` | FBXLoader |
| glTF/GLB | `.glb`, `.gltf` | GLTFLoader |
| OBJ | `.obj` | OBJLoader |

### 텍스처 (자동 감지)

| 포맷 | 확장자 |
|------|--------|
| PNG | `.png` |
| JPEG | `.jpg`, `.jpeg` |
| TGA | `.tga` |
| TIFF | `.tiff`, `.tif` |
| BMP | `.bmp` |
| EXR | `.exr` |

### 출력

| 유형 | 파일명 패턴 | 설명 |
|------|-------------|------|
| 썸네일 | `{모델명}_thumbnail.png` | 뷰포트 스크린샷 |
| 검증 리포트 | `{모델명}_report.html` | HTML 상세 검증 보고서 |

## 3. 드래그 앤 드롭

두 가지 드래그 앤 드롭 시스템이 동시에 동작합니다:

### Tauri 네이티브 드롭
- `appWindow.onDragDropEvent()` API 사용
- 운영체제 네이티브 파일 드롭 이벤트 처리
- 지원 확장자만 필터링 (fbx, glb, gltf, obj)

### HTML5 드롭존
- 브라우저 표준 `onDragOver` / `onDrop` 이벤트 처리
- 드래그 시 시각적 피드백 (테두리 강조, 스케일 애니메이션)
- 파일이 없을 때 "Drag & Drop 3D File" 안내 표시

### 파일 경로 처리
- Windows 백슬래시 경로(`C:\Users\...`)를 Tauri asset 프로토콜 URL로 자동 변환
- 한글, 공백, 특수문자가 포함된 경로 지원
- 경로 세그먼트별 개별 인코딩으로 슬래시 보존

## 4. 뷰 모드

### 4.1 Solid (기본)
- 기본 머티리얼 렌더링
- 내장 텍스처가 있으면 표시
- 환경 조명 (Studio 프리셋)

### 4.2 Wireframe
- 메시의 엣지 구조를 표시
- 원래 머티리얼을 어두운 반투명으로 교체
- 초록색 와이어프레임 라인 오버레이
- 토폴로지 구조 확인에 유용

### 4.3 Normals (노말 검사)
- 각 버텍스 위치에 점으로 노말 방향 시각화
- **파란 점 (작은 크기):** 정상 노말
- **빨간 점 (큰 크기):** 뒤집힌 노말
- 배경 메시를 어두운 반투명으로 교체하여 가독성 확보
- 뒤집힌 노말이 있으면 하단 중앙에 "Flipped Normals (개수)" 포커스 버튼 표시
- 버튼 클릭 시 뒤집힌 노말 중심 위치로 카메라 자동 이동

### 4.4 Normal Map (노말맵 뷰)
- 커스텀 셰이더로 버텍스 노말 방향을 RGB 색상으로 시각화
- X축 → R, Y축 → G, Z축 → B (범위 0~1 매핑)
- 전형적인 보라색/파란색 노말맵 렌더링

### 4.5 UV
- UV 체커 텍스처로 UV 펼침 상태를 시각화
- 빨간/남색 체크무늬 패턴 적용
- 그리드 라인 표시로 UV 왜곡 확인 가능
- UV 시임과 스트레칭 감지에 유용

### 4.6 Retopo (리토폴로지 진단)
- 삼각형 밀도 히트맵으로 메시 품질 시각화
- **색상 의미:**
  - 파란색: 과밀 영역 (평균 대비 삼각형이 너무 작음)
  - 초록색: 적정 밀도
  - 빨간색: 과소 영역 (삼각형이 너무 큼)
  - 노란색 계열: 얇은 삼각형 (aspect ratio 불량)
- **상단 진단 패널 표시:**
  - 리토폴로지 필요 여부 판정 (초록/빨간 표시등)
  - 총 삼각형 수
  - 얇은 삼각형 비율 (aspect ratio > 10이면 불량)
  - 과밀/과소 영역 비율
  - 밀도 편차 비율
  - 문제 사유 목록
  - 색상 범례

## 6. 검증 시스템

모델 로드 시 자동으로 6개 카테고리에서 검증을 수행합니다.

### 6.1 Geometry (지오메트리)

| 항목 | Good | Warning | Bad |
|------|------|---------|-----|
| 삼각형 수 (Tris) | < 100K | < 500K | >= 500K |
| 버텍스 수 (Verts) | < 100K | < 300K | >= 300K |
| 메시 수 | < 50 | < 100 | >= 100 |
| 파일 크기 | < 50MB | < 100MB | >= 100MB |
| 퇴화 삼각형 | < 1% | < 5% | >= 5% |
| 바운딩 박스 | 정보 표시 (X x Y x Z) | | |

### 6.2 Topology (토폴로지)

| 항목 | Good | Warning | Bad |
|------|------|---------|-----|
| 넌매니폴드 엣지 | 0 | <= 50 | > 50 |
| 열린 엣지 | 0 (방수) | 존재 시 경고 | |
| 뒤집힌 노말 | 없음 | 존재 시 | > 10% |

### 6.3 UV

| 항목 | Good | Warning | Bad |
|------|------|---------|-----|
| UV 커버리지 | 모든 메시에 UV 있음 | | UV 없는 메시 존재 |
| UV 채널 수 | 1~2개 채널 | | |

### 6.4 Texture (텍스처)

| 항목 | Good | Warning | Bad |
|------|------|---------|-----|
| 텍스처 수 | 정보 표시 | | |
| 누락 텍스처 | 0 | 1~2개 | >= 3개 |
| 최대 해상도 | < 4096px | < 8192px | >= 8192px |

### 6.5 Material (머티리얼)

| 항목 | Good | Warning | Bad |
|------|------|---------|-----|
| 머티리얼 수 | 정보 표시 | | |
| 머티리얼 미할당 | 없음 | 존재 시 경고 | |

### 6.6 Scale/Transform (스케일/트랜스폼)

| 항목 | Good | Warning | Bad |
|------|------|---------|-----|
| 비균일 스케일 | X=Y=Z | 불일치 시 경고 | |
| 피벗 오프셋 | < 1 유닛 | > 10 유닛 | |

### 종합 판정
- **Good** (초록): 모든 항목 통과
- **Warning** (노란): 주의 필요 항목 존재
- **Bad** (빨간): 심각한 문제 존재
- 전체 판정은 가장 나쁜 항목의 심각도를 따름

## 7. 텍스처 자동 감지

### 지원 텍스처 유형

| 유형 | 매칭 키워드 |
|------|-------------|
| BaseColor | basecolor, diffuse, albedo, color, col, diff |
| Normal | normal, nrm, norm, nml |
| Roughness | roughness, rough, rgh |
| Metallic | metallic, metalness, metal, mtl |
| AO | ao, ambientocclusion, occlusion, occ |
| Emissive | emissive, emission, emit |
| Height | height, displacement, disp, bump |
| Opacity | opacity, alpha, transparency |

### 매칭 규칙
- 대소문자 무시
- 파일명 끝에서 구분자(`_`, `-`, `.`) 뒤의 키워드 매칭
- 패턴: `[_\-.]<type>$` (확장자 제외 파일명 끝부분)
- 인식된 이미지 확장자만 처리
- 누락 텍스처 감지: 외부 텍스처가 존재할 때 basecolor, normal, roughness 확인

## 8. 내보내기 기능

### 썸네일 (Thumbnail)
- 현재 뷰포트를 PNG 스크린샷으로 저장
- 그리드는 캡처에서 자동 제외
- 모델 파일 옆에 `_thumbnail.png`로 저장
- 상태 피드백: 대기 → 저장 중 → 완료/실패

### 검증 리포트 (Report)
- HTML 형식의 상세 검증 보고서 생성
- 모델 파일 옆에 `_report.html`로 저장
- **포함 내용:**
  - 생성 타임스탬프 (한국어 로케일)
  - 파일 정보: 포맷, 파일명, 종합 판정
  - 카테고리별 검증 결과
  - 각 항목별 한국어 설명 및 기준치
  - 텍스처 파일 목록
  - 색상 코딩된 심각도 표시
  - 인쇄 친화적 스타일링

## 9. UI 구성

### 레이아웃

```
┌──────────────────────────────────────────────┐
│ [FBX] filename.fbx [Good]     │   Info Panel │
│                                │  (접기 가능) │
│                                │  - Geometry  │
│                                │  - Topology  │
│         3D 뷰포트              │  - UV        │
│    (OrbitControls)             │  - Texture   │
│                                │  - Material  │
│                                │  - Transform │
│                                │              │
│ [Solid][Wire][Normals]...      │ [Report][Thumb]
│ [Normal Map][UV][Retopo] [●●●] │              │
└──────────────────────────────────────────────┘
```

### 상단 좌측
- 파일 포맷 뱃지 (FBX, GLB 등)
- 파일명 (긴 이름은 truncate)
- 검증 상태 뱃지 (Good/Warning/Bad)

### 우측 패널
- 접기/펼치기 가능한 상세 정보 패널
- 카테고리별 그룹화된 검증 결과
- 각 항목별 색상 코딩
- 텍스처 파일 목록 (유형 + 파일명)
- 스크롤 가능

### 하단 좌측
- 뷰 모드 전환 버튼 (6개)
- 배경색 선택기 (3종: 어둡게, 중간, 밝게)
- 반응형: 창이 좁으면 자동 줄바꿈

### 하단 우측
- Report 생성 버튼
- Thumbnail 저장 버튼

### 스타일 테마
- **배경:** 반투명 어두운 파란 (`rgba(16, 24, 48, 0.94)`)
- **백드롭 필터:** `blur(20px)` (프로스트 글래스 효과)
- **액센트 컬러:** `#e94560` (코랄 레드)
- **성공:** `#4ade80` (초록)
- **경고:** `#fbbf24` (노란)
- **오류:** `#f87171` (빨간)

## 10. 백엔드 명령어 (Tauri)

### `scan_asset_directory`
- **입력:** 모델 파일 경로
- **기능:** 모델 파일이 위치한 디렉토리 스캔
- **출력:** 모델 메타데이터, 형제 파일 목록, 발견된 텍스처 파일 목록
- 이미지 확장자만 텍스처로 분류

### `save_thumbnail`
- **입력:** base64 PNG 데이터, 저장 경로
- **기능:** base64 디코딩 후 PNG 파일 저장
- 상위 디렉토리 자동 생성

### `save_text_file`
- **입력:** 텍스트 콘텐츠, 저장 경로
- **기능:** HTML 리포트 파일 저장
- 상위 디렉토리 자동 생성
